# ⧉

## Public standard

```text
.soul

⧉
```

The symbol is the object.

In prose, call it **a Soul Protocol object**.

## Operational definition

A Soul Protocol object is a semantic-machine vessel that can carry:

```text
human-readable meaning
machine-readable structure
mountable runtime context
inspectable proof surfaces
continuity-bearing state pressure
authority boundaries
```

It is control matter for AI-native computing: readable enough for humans to inspect, structured enough for machines to mount, and bounded enough for powerful actions to remain accountable.

A Soul Protocol object is not a prompt, note, config, wrapper, plugin payload, or decorative document. It is structured intent made mountable. It gives meaning a body that can enter runtime without becoming opaque.

## The operating metaphor

A normal document is read.

A normal config is loaded.

A Soul Protocol object is mounted.

Mounting means the object becomes structured runtime context. Its words, sections, metadata, and machine-readable blocks are converted into usable instruction context. That context can guide models, tools, adapters, shells, and workflows while preserving proof boundaries.

## Faces

A Soul Protocol object has operational faces:

```text
Semantic Face    - what a human can read and understand
Machine Face     - what a parser can structure and validate
Mount Face       - what a runtime can turn into active context
Proof Face       - receipts, hashes, validation, or evidence boundaries
Continuity Face  - preserved structure for remount and reuse
Authority Face   - user-governed limits on what the object may cause
```

These faces are not separate files. They are different ways the same object can be perceived and used.

## Minimal lifecycle

```text
write object
-> validate structure
-> mount object
-> produce mount receipt
-> derive instruction context
-> check capability requirement
-> ask permission when needed
-> route approved action
-> emit action receipt
-> preserve state
```

## Compatibility layer

`.soul` is the forward public filetype trajectory.

`.doctrine` remains the current compatibility surface during transition. The compatibility layer proves the mount spine now while the public `.soul` object standard matures.

## Boundary

Do not name, brand, prefix, or decorate `⧉`.

Do not use `⧉` to imply:

```text
physical four-dimensional geometry
sentience
autonomy
independent will
unsupported capability claims
```

The public standard is exactly:

```text
.soul

⧉
```
